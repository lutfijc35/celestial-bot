✦ Celestial Bot — Setup Guide
📋 Overview 🔧 Discord Setup ⚙️ Environment 🚀 Jalankan Bot 🛡️ Admin Commands ✅ Verifikasi
📋 Overview
Ringkasan bot, tech stack, dan urutan setup dari awal hingga siap digunakan.

Celestial adalah Discord bot untuk manajemen guild game Epic Seven — mengelola registrasi member, assignment role otomatis, dan menampilkan informasi guild secara real-time. Bot beroperasi dengan slash commands dan modal form, mendukung dua mode approval: manual dan otomatis.

🔩 Tech Stack
Komponen Teknologi

Frameworkdiscord.py 2.x (app_commands)

Configpython-dotenv (.env)

🔀 Approval Mode

Dikonfigurasi via env var APPROVAL_MODE. Pilih satu mode sebelum menjalankan bot.

⏳ manual
  • • Setelah /register, akun masuk status pending
  • • Bot kirim embed ke #approval-request
  • • Admin klik ✅ Approve atau ❌ Reject
  • • Cocok untuk server yang ingin kontrol ketat
⚡ auto
  • • Setelah /register, akun langsung approved
  • • Role langsung di-assign tanpa perlu admin
  • • Guild list langsung ter-update
  • • Cocok untuk server yang ingin proses cepat

Atur permissions agar #register-here dan #other-games hanya terlihat setelah user mendapat role Member (via react ✅ di rules).

3. Cara Mendapatkan Discord IDs
💡 Aktifkan Developer Mode terlebih dahulu
  1. Buka Discord → Settings (⚙️) → Advanced
  2. Toggle Developer Mode → ON
  3. Kembali ke server — klik kanan pada channel, role, atau nama server
  4. Pilih Copy ID di bagian bawah menu konteks
  5. Paste ID tersebut ke nilai env var yang sesuai di .env
🚀 Jalankan Bot
Dua cara menjalankan bot: langsung via Python atau via Docker Compose.
Install dependencies & jalankan
pip install -r requirements.txt python main.py
💡
Gunakan virtual environment untuk menghindari konflik package:

python -m venv venv  →  source venv/bin/activate (Linux/Mac)
python -m venv venv  →  venv\Scripts\activate (Windows)
✅ Bot berhasil start jika log menampilkan: "Logged in as [NamaBot]#[xxxx]"
Build & jalankan container
# Pertama kali (build image + start container) docker-compose up --build -d # Selanjutnya (start tanpa rebuild) docker-compose up -d # Lihat log secara realtime docker-compose logs -f # Stop bot docker-compose down
⚠️
Pastikan file .env sudah diisi sebelum menjalankan Docker. Container membaca env vars dari file .env secara otomatis.
💡 Database SQLite tersimpan di ./data/ yang di-mount sebagai Docker volume. Data tidak hilang saat container restart atau rebuild.
🛡️ Admin Commands
Jalankan commands ini berurutan setelah bot online dan terhubung ke server.
1
Mapping Guild Roles — /set-guild

Buat mapping antara nama guild dan Discord role. Bot akan assign role ini ke user yang terdaftar di guild tersebut saat akun di-approve.

⚠️ Nama role Discord harus sama persis dengan nama guild yang dimasukkan user saat register.

Contoh: /set-guild @Covenants — mapping guild "Covenants" ke role @Covenants
Ulangi: Jalankan command ini untuk setiap guild yang ada di server
2
Set Info Guild — /guild-set-info

Isi informasi detail tiap guild: tipe kompetisi, level, dan keterangan. Informasi ini ditampilkan di #guild-list.

Jalankan /guild-set-info → Dropdown: pilih guild → Dropdown: pilih tipe → Modal: isi level + keterangan → Guild list auto-update
Tipe guild: casual  ·  semi_compe  ·  compe
Ulangi: Jalankan untuk setiap guild yang sudah di-mapping di Step 1
3
Post Rules Embed — /setup-rules

Jalankan command ini di dalam channel #rules. Bot akan memposting embed rules lengkap dengan tombol react ✅.

Message ID tersimpan otomatis ke database. Bot menggunakan ID ini untuk mendeteksi reaksi ✅ dari member (on_raw_reaction_add).

⚠️ Jalankan hanya satu kali. Jika dijalankan ulang, bot akan memposting embed baru dan Message ID lama tidak akan aktif untuk deteksi reaksi.
4
Post Register Embed — /setup-register

Jalankan command ini di dalam channel #register-here. Bot akan memposting embed dengan tombol "Daftar Sekarang" yang membuka modal register.

User yang sudah punya role Member (setelah react ✅ di rules) dapat mengklik tombol ini untuk memulai proses registrasi akun game.

5
Set Member List Channel — /setup-profile

Jalankan command ini di dalam channel yang ingin dijadikan member list (biasanya #member-list). Bot akan memposting embed daftar member terdaftar yang auto-update.

Step ini opsional — jalankan jika server ingin menampilkan daftar member secara publik. Channel yang dipilih akan menjadi channel member list permanen.

6
Set Welcome Channel — /setup-welcome

Jalankan command ini di dalam channel welcome (biasanya #welcome). Bot akan menyimpan channel tersebut sebagai tujuan pesan selamat datang anggota baru.

Step ini opsional — jika tidak diset (dan WELCOME_CHANNEL_ID di .env kosong), bot tidak akan mengirim pesan welcome saat member baru bergabung.

7
Set Register Channel — /setup-register-here

Jalankan command ini di dalam channel #register-here. Bot menyimpan channel tersebut sebagai register channel yang di-unlock otomatis setelah user react ✅ di rules.

Opsional — bisa juga diset via env var REGISTER_CHANNEL_ID. Jika diset via command, nilai di DB akan override env var.

8
Set Other Games Channel — /setup-other-games

Jalankan command ini di dalam channel #other-games. Bot menyimpan channel tersebut sebagai other-games channel yang di-unlock otomatis setelah user react ✅ di rules.

Opsional — bisa juga diset via env var OTHER_GAMES_CHANNEL_ID. Jika diset via command, nilai di DB akan override env var.

9
Set Approval Ping — /setup-approval-ping @role

Toggle role yang akan di-ping setiap ada approval request baru masuk. Bisa menambahkan lebih dari 1 role — jalankan command lagi dengan role berbeda untuk menambah, atau jalankan dengan role yang sama untuk menghapus (toggle).

Contoh: /setup-approval-ping @Admin — tambah @Admin ke daftar ping
/setup-approval-ping @Moderator — tambah @Moderator ke daftar ping
/setup-approval-ping @Admin — hapus @Admin dari daftar ping (toggle)

Opsional — hanya relevan jika APPROVAL_MODE=manual. Jika tidak ada role yang di-setup, approval request tetap dikirim tanpa ping.

⚡ Force Refresh & Admin Utilities (opsional)
/guild-info — Force refresh semua pesan guild list di #guild-list secara manual (edit semua halaman sekaligus)
/profile-list — Force refresh embed daftar member di channel member list
/guild-list — Lihat semua guild yang sudah di-mapping beserta role, tipe, level. Bisa langsung hapus dari dropdown
/remove-guild — Hapus mapping guild dari database (dropdown pilih guild). Guild list ter-update otomatis
/admin-unregister @user — Hapus akun game user tertentu tanpa perlu meminta user untuk unregister sendiri
/bot-status — Lihat uptime, latency, statistik akun, jumlah guild, dan konfigurasi channel aktif
/setup-changelog — Set channel ini sebagai changelog channel
/changelog — Post changelog dari git commits terbaru ke channel changelog. Otomatis track commits baru saja
/setup-starboard — Set channel ini sebagai starboard target (tempat repost pesan bintang)
/setup-starboard-source — Set channel ini sebagai sumber starboard (pesan yang dimonitor untuk react ⭐)
/setup-starboard-role @role — Set role sementara yang diberikan saat pesan masuk starboard (otomatis dicabut setelah 30 hari)
/setup-starboard-threshold <count> — Set minimum bintang untuk masuk starboard (default: 5)
/leaderboard — Top 10 user dengan pesan paling banyak di starboard (bisa dijalankan semua user)
/help — Lihat daftar semua command bot beserta syntax (bisa dijalankan semua user)
✅ Verifikasi
Test semua flow ini setelah setup selesai untuk memastikan bot berfungsi normal.
✅ Welcome message — User baru join server → bot kirim embed welcome di #welcome secara otomatis
✅ React rules — User react ✅ di pesan rules → mendapat role Member, channel #register-here dan #other-games menjadi visible
✅ Modal register — User klik tombol "Daftar Sekarang" di #register-here → modal register terbuka dengan 3 field: Server, Guild (opsional), Nickname
✅ Submit register — User submit modal → embed approval muncul di #approval-request manual mode atau langsung approved auto mode
✅ Approve & role assignment — Admin klik ✅ Approve → role guild di-assign ke user, semua pesan di #guild-list ter-update otomatis (member tampil horizontal)
✅ Profile command — Jalankan /profile → embed profil akun user tampil dengan data yang benar
✅ Admin unregister — Jalankan /admin-unregister @user → akun terhapus dari database, #guild-list dan #member-list ter-update otomatis
✅ Approval ping — Jalankan /setup-approval-ping @Admin → user register baru → embed approval di #approval-request mention @Admin
✅ Remove guild — Jalankan /remove-guild → pilih guild → guild terhapus dari database, #guild-list ter-update otomatis
✅ Starboard — Setup 4 command starboard → kirim pesan di source channel → react ⭐ x5 → embed muncul di target channel + role sementara di-assign ke author
✅ Leaderboard — Jalankan /leaderboard → embed ranking top 10 user starboard tampil
